Diagnóstico completo de por que as imagens não estavam chegando:

### Bug 1 - apiIntegrationTool.ts: Truncamento do base64
**Problema:** `.substring(0, 5000)` cortava o base64 da imagem em ~5KB, corrompendo os dados
**Correção:** Aumentado para `.substring(0, 100000000)` (100MB) — base64 de imagens pode ter vários MB

### Bug 2 - fileSystemTool.ts: Escrita como UTF-8 em vez de binário
**Problema:** `fs.writeFileSync(path, content, "utf-8")` corrompia dados binários de imagem
**Correção:** Adicionada detecção inteligente:
- Data URLs (`data:image/png;base64,...`) → decodifica base64 e escreve como Buffer
- Base64 puro → decodifica e escreve como Buffer
- Extensões de imagem (.png, .jpg, etc.) → tenta escrita binária primeiro
- Texto normal → UTF-8 como antes

### Bug 3 - telegram.ts: Nunca enviava a foto
**Problema:** A resposta do orquestrador (texto sobre a imagem) era enviada apenas como `sendMessage()`, nunca como `sendPhoto()`
**Correção:** Adicionada função `findGeneratedImagePath()` que:
1. Procura por caminhos `generated_images/...` na resposta textual
2. Verifica se o arquivo existe no disco
3. Fallback: busca arquivos recentes na pasta `generated_images/`
4. Se encontrou, envia via `sendPhoto()` com a legenda da resposta
5. Se não encontrou, envia como texto normal (comportamento anterior)
