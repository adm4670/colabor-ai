1. Visao Geral

1.1 O que e o colabor-ai?

O colabor-ai e uma plataforma open-source para criar e orquestrar agentes de IA especializados que colaboram entre si para resolver tarefas. Imagine uma equipe de mini-especialistas em IA, cada um excelente em uma area diferente, coordenados por um maestro inteligente.

Analogia: Assim como uma orquestra tem violinistas, pianistas e percussionistas, cada um tocando sua parte sob a regencia de um maestro, o colabor-ai tem agentes especializados (Chat, Python, FileSystem, etc.) orquestrados por um Planner que decide quem faz o que e em qual ordem.
AgenteEspecialidadeOnde Roda
PlannerDecidir qual agente deve agir e com qual instrucaoCloud
AssistantConversa geral, explicacoes, duvidasCloud
PythonCalculos, analise de dados, execucao de codigoCloud
WriterFormatacao da resposta final para o usuarioCloud
File SystemLer, criar, editar e organizar arquivos no PCClient (Windows)
ShellExecutar comandos CMD e PowerShellClient (Windows)
DesktopScreenshot, clipboard, processos e janelasClient (Windows)

1.2 Filosofia do Projeto

"Be genuinely helpful, not performatively helpful. Skip the 'Great question!' and 'I'd be happy to help!' — just help. Actions speak louder than filler words."
  • Modularidade: cada agente e independente e pode ser substituido ou melhorado isoladamente
  • Seguranca: ferramentas perigosas exigem confirmacao; Cloud e Client sao isolados
  • Extensibilidade: novas ferramentas e agentes podem ser adicionados sem alterar o nucleo
  • Transparencia: streaming de eventos permite que o usuario veja o progresso em tempo real

3. Modulo Core (Engine)

O Core e a biblioteca compartilhada que contem as classes fundamentais usadas por todos os modulos. Esta na raiz do projeto (core/) e define a base para agentes, orquestradores e ferramentas.

3.1 Agent (core/agent/agent.ts)

Cada agente e um wrapper sobre um modelo de linguagem (OpenAI/DeepSeek) que encapsula: personalidade (nome, papel, objetivo, backstory), historico de conversa (contexto da sessao), tools (ferramentas que pode chamar) e system prompt (instrucoes de comportamento).

Loop de Tool Calling: Quando o Agent detecta que o LLM retornou tool_calls, ele executa as funcoes registradas, adiciona os resultados ao historico, e reenvia para o LLM — repetindo ate que o modelo produza uma resposta textual final. 
const agent = new Agent({
      name: "MathAgent",
      role: "Math expert",
      goal: "Solve mathematical problems",
      backstory: "An AI specialized in calculations.",
      model: "gpt-5-nano",
      tools: [pythonExecTool],
      functions: { execute_python: pythonExecTool.handler }
    });
    const resposta = await agent.run("Quanto e 352 x 127?");

3.2 AgentOrchestrator (core/orchestrator/orchestrator.ts)

O AgentOrchestrator e o maestro que coordena multiplos agentes:

+-------------------------------------+ | ORCHESTRATOR LOOP | | | | 1. Planner decide o que fazer | | 2. Planner escolhe um agente | | 3. Agente executa a tarefa | | 4. Resultado e registrado | | 5. Planner decide: continuar | | ou "finish"? | | 6. Se finish -> responde! | | Se nao -> volta ao passo 1 | | | | (max. 10 iteracoes, protecao | | contra loops infinitos) | +-------------------------------------+

O Planner responde sempre em JSON:

// Precisa de acao:
    { "agent": "python_code", "instruction": "Calcule 352 x 127 usando Python" }
    
    // Satisfeito:
    { "agent": "finish", "instruction": "O resultado e 44.704" }

O Cloud e o cerebro do assistente. Ele pensa, decide e coordena tudo. O app (Client) so mostra o resultado.

4.3 Memory Engine

Sistema de memoria de longo prazo com:

  • MEMORY.md: fatos permanentes que nunca sao esquecidos
  • Notas diarias (memory/): registros automaticos de cada conversa
  • Busca por similaridade: tokenizacao + scoring para encontrar memorias relevantes
  • Cache TTL (10s): evita leitura repetida de arquivos
  • Consolidacao: extrai fatos e decisoes importantes das conversas

4.4 Context Engine

Gerencia o orcamento de tokens do LLM com estrategia hibrida:

ParametroValorDescricao
maxTokens8000Limite maximo de tokens no contexto
recentRatio0.6 (60%)Proporcao reservada para mensagens recentes
keepRecentIntact5Ultimas N mensagens nunca sao resumidas
modesummarizeUsa LLM para resumir em vez de truncar
Exemplo: Numa conversa de 20 mensagens que excede 8000 tokens, o Context Engine mantem as 5 ultimas intactas, resume as 15 anteriores em um paragrafo usando o LLM.

4.5 Autenticacao JWT

  • Login: aceita API key ou cria sessao anonima
  • Token JWT: assinado com HMAC-SHA256, expira em 24h
  • Middleware: protege todas as rotas /chat/*
  • Refresh: renova token sem reautenticar
  • WebSocket: token passado como query param ?token=...

8. Guia de Inicio Rapido

Pre-requisitos

  • Node.js 18+
  • Python 3.x (para execute_python tool)
  • API Key DeepSeek ou OpenAI

Instalacao

git clone https://github.com/adm4670/colabor-ai.git
    cd colabor-ai
    npm install
    cd cloud && npm install && cd ..
    cd client && npm install && cd ..

Configuracao

# .env na raiz (Core)
    OPENAI_API_KEY=sk-...
    TELEGRAM_TOKEN=123456:ABC...
    
    # cloud/.env
    DEEPSEEK_API_KEY=sk-...
    LLM_PROVIDER=deepseek
    JWT_SECRET=seu-segredo
    PORT=3001

Executando

# Core: agente unico no terminal
    npm run start:dev:agent
    
    # Core: multi-agente no terminal
    npm run start:dev:orchestrator
    
    # Cloud: servidor backend
    cd cloud && npm run dev
    
    # Client: app Electron (modo dev)
    cd client && npm run dev

Testes

npm test           # Core
    cd cloud && npm test  # Cloud (42 testes)
