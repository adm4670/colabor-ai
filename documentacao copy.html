01 Visao Geral

O que e?

colabor-ai e uma plataforma Node.js/TypeScript para criar e executar agentes de IA que colaboram entre si. O sistema usa o padrao Planner-Worker: um agente planejador decide qual agente especialista executar, e um agente refletor avalia os resultados.

Objetivos

  • Fornecer runtime minimo para agentes de IA
  • Permitir ferramentas e integracoes modulares
  • Suportar workflows com agentes colaborativos
  • Servir como base para o ecossistema colabor.ai

Stack Tecnologica

Node.js v23
TypeScript 5.3+
DeepSeek API

Padrao Planner-Worker com Reflection

        Usuario → Orchestrator → Planner ("qual agente?")
                                       ↓
                                Agent selecionado executa
                                       ↓
                                Reflector avalia resultado
                                (success/complete/learning)
                                       ↓
                                Loop ate "finish" ou max steps
                                       ↓
                                AnswerAgent → Resposta final

03 Fluxo de Execucao

1. Entrada do Usuario

CLI (main.ts) ou Telegram Bot (telegram.ts) recebem input. O AgentRegistry fornece o planner e sub-agentes registrados.

2. Carregamento de Contexto

Transcript da sessao e carregado do JSONL versionado. MemoryEngine busca fatos relevantes no MEMORY.md e notas diarias. Rate limit e verificado.

3. Loop Planner-Worker (max 10 passos)

O PlannerAgent decide qual agente executar (JSON: {"agent":"...","instruction":"..."}). O agente selecionado executa com suas tools. O ContextEngine gerencia o budget de tokens. Detecta instrucoes repetidas para evitar loops.

4. Reflection Step (diferencial)

O ReflectorAgent (agente separado, modelo leve) avalia: o agente teve sucesso? O resultado esta completo? Deveriamos tentar abordagem diferente? O que aprendemos?

5. Finalizacao

Quando o Planner retorna "finish", o AnswerAgent formata a resposta. Transcript salvo, nota diaria criada, memoria consolidada.

04 Agentes (8)

Planner

PlannerAgent

Decide qual agente deve executar o proximo passo. Usa deepseek-v4-pro. Retorna JSON com agent e instruction.

NOVO

ReflectorAgent

Avalia resultados dos agentes. Usa modelo leve (deepseek-chat via REFLECTOR_MODEL). Retorna {success, complete, missingInfo, retryDifferent, learning}.

Chat

AssistantAgent

Conversacao geral, explicacoes, perguntas simples. Tem acesso ao memory_search tool.

Code

PythonAgent

Executa codigo Python com timeout configuravel (PYTHON_TIMEOUT). Retorna RESULT + DETAILS.

ShellAgent

Executa comandos shell (npm, git, mkdir). Protecao contra comandos perigosos (rm -rf /, shutdown).

BrowserAgent

Automacao web via Puppeteer. Navega, clica, preenche forms, extrai texto, screenshots.

AnswerAgent (WriterAgent)

Transforma saidas tecnicas em mensagens naturais para WhatsApp/Telegram. Remove ruido tecnico.

TaskManagerAgent

CRUD de tarefas/atividades. Persiste em tasks.json com create_activity, get_activities_by_day, delete_activity.

07 Agent Registry Centralizado

Antes vs Depois

ANTES (main.ts)

import { plannerAgent } from "../agents/planner.agent";
    import { pythonAgent } from "../agents/python.agent";
    import { answerAgent } from "../agents/answer.agent";
    // ... +5 imports diretos
    
    const orchestrator = new AgentOrchestrator(planner, [
      { name: assistantAgent.name, ... },
      { name: pythonAgent.name, ... },
      // ... +5 entradas manuais
    ]);

DEPOIS (main.ts)

import { agentRegistry } from "../agents/agent-registry";
    // 1 import, sem listas manuais!
    
    const orchestrator = new AgentOrchestrator(
      agentRegistry.getPlanner(),
      agentRegistry.getSubAgents()
    );

Metodos principais

  • register(entry) — registra agente
  • registerPlanner(agent) — registra planner
  • get(name) / find(name) — busca exata / case-insensitive
  • getSubAgents() — formato esperado pelo Orchestrator
  • getAgentListForPrompt() — formatado para o planner
  • listNames() / size — listagem

Auto-Registro

Cada agente se registra no final do seu modulo:

// No final de python.agent.ts:
    agentRegistry.register({
      name: pythonAgent.name,
      description: "Python execution specialist...",
      agent: pythonAgent,
      role: "PythonAgent",
      useWhen: ["calculations", "data analysis"]
    });

Diferenciais do colabor-ai

O colabor-ai nao compete em escala com o openclaw. Seus diferenciais sao: Planner-Worker (decide qual agente chamar baseado na intencao) e Reflection (auto-avaliacao dos resultados). Nenhum dos dois existe no openclaw.

09 Melhorias Implementadas

PRIORIDADE 1

Formato SKILL.md com YAML Frontmatter

Adotado o formato do OpenClaw: frontmatter YAML com name, description, keywords. Parser inline sem dependencias externas. Fallback para formato antigo.

Arquivos: skills/*.md, core/skills/skills-manager.ts, core/skills/skills-manager.spec.ts
PRIORIDADE 2

JSONL de Sessao Versionado (v1)

Header {"type":"session","version":1,"id":"...","timestamp":"..."} na primeira linha. Backward compat com arquivos antigos. Migracao automatica. Suporte a parentSession.

Arquivos: core/session/transcript.ts, core/session/session.test.ts
PRIORIDADE 3

Agent Registry Centralizado

Classe AgentRegistry com register(), get(), find(), getSubAgents(). Cada agente se registra no proprio modulo. main.ts e telegram.ts usam o registry em vez de imports diretos.

Arquivos: core/agents/agent-registry.ts, core/agents/agent-registry.spec.ts, core/orchestrator/main.ts, core/orchestrator/telegram.ts + 7 agentes
PRIORIDADE 4

Reflection Separado do Planning

ReflectorAgent dedicado avalia resultados (deepseek-chat via REFLECTOR_MODEL). O Planner agora so decide qual agente executar. Separacao de concerns: planejar != avaliar.

Arquivos: core/agents/reflector.agent.ts, core/orchestrator/orchestrator.ts
