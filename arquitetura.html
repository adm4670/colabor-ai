☁️ Cloud — Núcleo de Inteligência

Responsável por toda a inteligência do sistema. Orchestrador, Agents, memória vetorial e comunicação com os clients Edge.

🚪

API Gateway

  • ▸ Express/Fastify com middleware de autenticação JWT
  • ▸ Rate limiting por usuário (Redis-based)
  • ▸ Rotas: /auth/*, /agents/*, /tools/*
🔐

Auth Google OAuth2.0

  • ▸ Login via Google com Passport.js
  • ▸ JWT com sub = userId, expiração 24h
  • ▸ Refresh token automático
🧠

Agent Engine

  • ▸ Planner — decide qual agente chamar baseado na mensagem
  • ▸ Orchestrator — gerencia o fluxo, reflexão, memória
  • ▸ 5+ Agents — DataAnalyst, Git, DB, Image, Notification
  • ▸ LLM: OpenAI GPT-4o / DeepSeek v4 (configurável)
📡

WebSocket Server

  • ▸ Socket.IO ou ws nativo
  • ▸ Mensagens: tool_call, tool_result, agent_response
  • ▸ Autenticação JWT na conexão
  • ▸ Heartbeat a cada 30s
🗄️

PostgreSQL + Redis + ChromaDB

  • ▸ PostgreSQL: users, sessions, conversations, configs
  • ▸ ChromaDB: 1 collection por usuário (user_{id}_memories)
  • ▸ Redis: cache de sessão, pub/sub para WebSocket, fila de tools
📂

💻 Edge — Execução na Máquina do Usuário

Aplicação desktop Electron que conecta à Cloud via WebSocket, executa ferramentas localmente e gerencia a comunicação bidirecional.

🖥️

Electron App

  • ▸ Interface React com chat, histórico e configurações
  • ▸ Tray icon + notificações nativas
  • ▸ Login via Google (webview embutida)
  • ▸ Instalador: .exe (Windows), .dmg (Mac), .deb (Linux)
🔧

Tool Executor Engine

  • ▸ FileSystem: read, write, list, delete, mkdir, copy, move
  • ▸ ShellExec: comandos no terminal (npm, git, etc)
  • ▸ PythonExec: executa scripts Python via python-shell
  • ▸ Browser: automação web via Puppeteer/Playwright
  • ▸ Image: captura de tela, análise de imagem local
🔗

WebSocket Client

  • ▸ Conexão persistente com a Cloud
  • ▸ Reconexão automática com backoff exponencial
  • ▸ Mensagens: tool_result, tool_error, heartbeat
🟢

Runtimes Embarcados

  • ▸ Node.js: embarcado via electron-builder (versão LTS)
  • ▸ Python: distribuição embedded portátil na pasta python/
  • ▸ Nenhuma instalação externa necessária pelo usuário
📂

🔄 Fluxo de Requisição — Passo a Passo

Como uma requisição do usuário percorre a arquitetura da Edge até a Cloud e volta.

1
💬Usuário envia mensagem

"Crie um relatório com esses arquivos da pasta Downloads"

Electron App→WebSocket Client
2
📡Edge envia para Cloud via WebSocket

Mensagem user_message com userId, sessionId e texto

3
🧠Cloud: Agent Engine processa

Planner decide: DataAnalystAgent é o melhor para a tarefa

4
🤖Agente precisa de uma Tool

DataAnalystAgent precisa ler arquivos → emite comando tool_call para fileSystem.read

5
📤Cloud envia Tool Call para Edge

WebSocket Server → Edge: { type: "tool_call", toolName: "fileSystem.read", params: { target: "~/Downloads" } }

6
⚡Edge executa Tool localmente

Tool Executor Engine roda fs.readdirSync no sistema do usuário

7
📥Edge envia resultado de volta

WebSocket Client → Cloud: { type: "tool_result", toolName: "fileSystem.read", result: {...} }

8
🔄Cloud processa e repete se necessário

Agente analisa resultado, decide próximos passos. Pode chamar mais tools ou finalizar.

9
✅Resposta final volta para o usuário

Cloud envia agent_response → Edge exibe no chat → Usuário vê o relatório pronto

📋 Protocolo WebSocket

Cloud → Edge

interface CloudToEdge {
      type: "tool_call"
         | "notification"
         | "agent_response"
      userId: string
      sessionId: string
      data: {
        toolName: string
        requestId: string
        params: any
      }
    }

Edge → Cloud

interface EdgeToCloud {
      type: "tool_result"
         | "tool_error"
         | "auth"
         | "heartbeat"
      userId: string
      sessionId: string
      data: {
        toolName: string
        requestId: string
        result?: any
        error?: string
      }
    }

🔒 Segurança — Isolamento Multi-tenant

Cada camada da arquitetura isola dados entre usuários para garantir que não haja vazamento de informação.

Camada Tecnologia Como isola
🗄️ Banco Relacional PostgreSQL Toda tabela tem FK user_id — consultas sempre filtradas por WHERE user_id = ?
🧬 Banco Vetorial ChromaDB Uma collection por usuário — nome: user_{id}_memories
⚡ Cache Redis Chaves prefixadas com user:{id}:*
📡 WebSocket Socket.IO Autenticação JWT na conexão — servidor só encaminha mensagens para o userId correspondente
🔧 Tools Tool Executor Cada tool roda no contexto do sistema daquele usuário (isolamento por processo)
🔐 Sessão JWT + Redis Token JWT com sub = userId, expira 24h, refresh token automático

📦 Instalador Electron

Configuração do electron-builder para gerar instaladores nativos com Node.js e Python embarcados.

⚙️

electron-builder.yml

appId: com.colabor-ai.desktop
    productName: colabor-ai
    directories:
      output: dist
      buildResources: build
    
    extraResources:
      - from: python/
        to: python/
        filter:
          - "**/*"
    
    files:
      - "**/*"
      - "!python/**/*"
    
    extraMetadata:
      main: build/main.js
    
    win:
      target:
        - target: nsis
          arch: [x64]
      extraResources:
        - from: python/embedded/
          to: python/
    
    mac:
      target:
        - target: dmg
          arch: [x64, arm64]
    
    nsis:
      oneClick: false
      allowToChangeInstallationDirectory: true
      installerIcon: build/icon.ico
